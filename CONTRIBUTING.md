# Contributing

Thank you for your interest in contributing to TaskFlow.

For project information, see the [README](README.md).

## Workflow

1. Create a new branch from `main`.
2. Implement the requested changes.
3. Run:

```bash
flutter format .
flutter analyze
flutter test
```

4. Commit using Conventional Commits.
5. Open a Pull Request.

## Commit Convention

Examples:

```text
feat: add authentication screen
fix: resolve login validation
docs: update README
chore: configure CI
```

## Pull Requests

Every Pull Request should include:

- Summary
- Changes
- Related Issue
